# Session Logger

## 概要

セッション中の会話内容（開始〜スキル起動時点）を構造化して要約し、マークダウンファイルとして保存するスキル。

## 起動タイミング

- ユーザーが `/session-logger` を実行したとき
- ユーザーが「セッションの内容をまとめて」「今日の作業を記録して」等と依頼したとき

## 出力先

`C:\development\session-logs\` ディレクトリに保存する。
ディレクトリが存在しない場合は、親ディレクトリも含めて再帰的に作成する（`mkdir -p` 相当）。

## ファイル名

`YYYY-MM-DD_HHmm_簡潔なトピック名.md` 形式で生成する。

- 日時はスキル起動時点のローカル時刻
- トピック名はセッションの主要テーマを英語で簡潔に（例: `youtube-discord-notifier-setup`）

## 出力フォーマット

以下の構造でマークダウンファイルを作成する：

```markdown
# セッションログ: （主要トピック）

**日時**: YYYY-MM-DD HH:MM
**作業ディレクトリ**: （該当パス）

## 概要

（セッション全体を2〜3文で要約）

## 実施内容

（時系列で主要な作業をリスト化。各項目は簡潔に）

- 作業1の内容
- 作業2の内容
- ...

## 作成・変更されたファイル

（新規作成や変更を行ったファイルのパスをリスト化）

- `path/to/file1` — 説明
- `path/to/file2` — 説明

## 技術的な決定事項

（セッション中に行われた重要な技術的判断をリスト化。なければ省略）

- 決定1: 理由
- 決定2: 理由

## 次のステップ

（未完了のタスクや今後の作業があれば記載。なければ省略）

- TODO項目
```

## 実行手順

1. セッション開始からスキル起動時点までの会話全体を振り返る
2. 上記フォーマットに従い内容を構造化する
3. 出力先ディレクトリの存在を確認し、なければ親ディレクトリも含めて再帰的に作成する
4. ファイルを書き出す
5. 保存先のパスをユーザーに通知する

## 注意事項

- APIキーやWebhook URL等の機密情報はログに含めない
- コードの全文は記載しない。何を作成・変更したかの要約にとどめる
- 日本語で記述する
- 簡潔さを優先し、冗長な説明は避ける
